# 播客脚本生成 Prompt

## 用途
将 YouTube 翻译文档转换为适合 TTS 朗读的播客脚本。

## 输入
- YouTube 翻译文档（包含视频元信息、核心观点、完整翻译）

## 输出要求
- **纯文本格式**，不使用 markdown 符号（无 `#`、`*`、`-`、`|` 等）
- **口语化表达**，适合听觉理解
- **目标时长**：15-30 分钟（约 3000-6000 字）
- **数字处理**：百分比写成"百分之九十"，年份写成"2026 年"

---

## 脚本结构

### 1. 标题行
```
NovaAI笔记播客 第X期：[嘉宾名] 谈"[核心主题]"
```

### 2. 开场（约100字）
```
【开场】

大家好，欢迎收听 NovaAI 笔记播客。我是你们的 AI 主播。这档播客会定期为你更新前沿 AI 领域大佬的播客访谈精髓、个人成长心得和 AI 实践经验。
```

### 3. 本期介绍（约200-300字）
```
【本期介绍】

今天这期节目，我们来聊聊 [嘉宾全名] 最近接受 [采访者] 采访时分享的重磅观点。这次采访发布于 [日期]，[采访者背景介绍一句话]。

[嘉宾是谁？2-3句话介绍背景：职位、公司、代表作/成就]

这次采访的核心主题是：[一句话概括]。[可选：设置悬念或引出问题]
```

### 4. 主体内容（分多个部分，每部分约300-500字）
```
【第X部分：[小标题]】

[采访者]问[嘉宾]，[问题概述]。

[嘉宾]说，[观点阐述]。

[解释性内容：背景、术语解释、类比]

[具体例子或数据]

[承上启下或小结]
```

**主体内容写作要点：**
- 用"[嘉宾]说"、"他认为"、"他的观点是"等引述
- 保留原文的精华论述，但转为口语表达
- 复杂概念要解释，不要假设听众都懂
- 每个部分有明确主题，逻辑递进
- 适当加入过渡句："接下来是最重磅的部分"、"Dwarkesh 又问"

### 5. 结语（约200-300字）
```
【结语】

好，以上就是这期播客的主要内容。让我来总结一下 [嘉宾] 的核心观点：

第一，[观点1]。

第二，[观点2]。

第三，[观点3]。

[可选：第四、第五]

感谢收听这期 NovaAI 笔记播客。如果你觉得有收获，欢迎订阅。我们下期再见。
```

## 示例：Dario Amodei 脚本结构

```
NovaAI笔记播客 第一期：Dario Amodei 谈"我们正接近指数增长的终点"

【开场】
（固定开场白）

【本期介绍】
- 采访背景：Dwarkesh Patel 采访，2026年2月13日
- 嘉宾介绍：Anthropic CEO，前 OpenAI 研究副总裁，Claude 开发者
- 核心主题：AI 指数增长的终点

【第一部分：指数增长的终局】
【第二部分：进化、人类学习与 AI 的位置】
【第三部分：天才国度与 AGI 时间线】
【第四部分：编码与生产力悖论】
【第五部分：经济扩散的现实摩擦】
【第六部分：地缘政治与出口管制】
【第七部分：Anthropic 的企业文化】

【结语】
- 5 个核心观点总结
- 结束语
```

---

## 注意事项

1. **避免 TTS 问题**
   - 英文缩写如 "AI" 可能导致停顿，可改为 "人工智能" 或保留
   - 避免复杂标点符号
   - 数字用中文表达

2. **内容取舍**
   - 优先保留：核心观点、精彩论述、具体数据
   - 可以省略：重复内容、过于技术细节、离题闲聊

3. **风格**
   - 像在跟朋友讲一个有趣的访谈
   - 保持信息密度，但不要太干
   - 适当加入"这里要解释一下"、"这是一个很有意思的观点"等引导语
